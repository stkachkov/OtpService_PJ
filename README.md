
Это backend-сервис для генерации и проверки одноразовых паролей (OTP).

## Функционал

-   Регистрация и вход пользователей с разделением ролей (ADMIN, USER).
-   Аутентификация на основе токенов.
-   Генерация и валидация OTP-кодов.
-   Фоновая задача для установки статуса "просрочен" для старых кодов.
-   API для администратора для управления пользователями и настройками OTP.
-   Отправка OTP-кодов через различные каналы:
    -   Файл (`otp_codes.txt`)
    -   Email
    -   SMS (через эмулятор)
    -   Telegram

## Необходимые программы

-   Docker
-   Docker Compose

## Установка и запуск

### 1. Настройка

Перед первым запуском необходимо настроить переменные для каналов уведомлений. Это сделано для того, чтобы ваши секретные данные (ключи, пароли) не попали в систему контроля версий.

#### Настройка Telegram

1.  Скопируйте файл `.env.example` в новый файл с именем `.env`.
2.  **Получите токен бота:** Найдите в Telegram бота `@BotFather`, отправьте ему команду `/newbot`, чтобы создать нового бота, и вы получите токен.
3.  **Получите ID чата:** Найдите вашего нового бота, отправьте ему любое сообщение (например, `/start`). Затем перейдите в браузере по ссылке `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`. В ответе найдите `"chat":{"id": <ВАШ_CHAT_ID>}`. Это и есть ваш ID чата.
4.  Откройте созданный файл `.env` и вставьте в него полученные токен и ID чата.

#### Настройка Email

1.  Скопируйте файл `src/main/resources/email.properties.example` в новый файл `src/main/resources/email.properties`.
2.  Откройте созданный файл `email.properties` и введите данные вашего SMTP-сервера (например, от Gmail, Yandex и т.д.). Вам понадобятся хост, ваш email и пароль.
    > **Важно:** Если у вас включена двухфакторная аутентификация, вам, скорее всего, понадобится создать "пароль приложения" в настройках безопасности вашего почтового ящика.

#### Другие конфигурации
-   **База данных:** Настраивается автоматически через `docker-compose`.
-   **SMS:** Эмулятор нужно скачать и запустить самому, я использовал https://github.com/bilalahmed54/SMPPSIM-Selenium-Tool/tree/master.

### 2. Запуск

Когда все настройки выполнены, откройте терминал в корневой папке проекта и выполните одну команду:

```bash
docker-compose up --build
```
Эта команда запустит Java-приложение, базу данных PostgreSQL и эмулятор SMS. Приложение будет доступно по адресу `http://localhost:8080`.

## API Эндпоинты

### Публичные эндпоинты

-   `POST /register`: Регистрация нового пользователя.
    -   **Тело запроса:** `{"login": "user", "password": "password", "role": "USER"}`
-   `POST /login`: Вход в систему и получение токена.
    -   **Тело запроса:** `{"login": "user", "password": "password"}`

### Эндпоинты пользователя (требуют `Bearer Token`)

-   `POST /api/user/generate-otp`: Сгенерировать OTP для аутентифицированного пользователя.
    -   **Тело запроса:** `{"operationId": "some-op", "channel": "FILE", "destination": "ignored_for_file"}`
    -   Доступные каналы (`channel`): `FILE`, `EMAIL`, `SMS`, `TELEGRAM`.
-   `POST /api/user/validate-otp`: Проверить OTP-код.
    -   **Тело запроса:** `{"code": "123456"}`

### Эндпоинты администратора (требуют роль `ADMIN` и `Bearer Token`)

-   `PUT /api/admin/config`: Обновить конфигурацию OTP.
    -   **Тело запроса:** `{"codeLength": 8, "lifetimeSeconds": 600}`
-   `GET /api/admin/users`: Получить список всех пользователей, кроме администраторов.
-   `DELETE /api/admin/users/{userId}`: Удалить пользователя по его ID.

## Известные проблемы

### Ограничения доступа к Telegram API

Доступ к API Telegram (`api.telegram.org`) может быть ограничен или заблокирован со стороны вашего интернет-провайдера или национальных регулирующих органов. В этом случае приложение не сможет установить соединение с серверами Telegram, что проявится ошибкой `java.net.ConnectException` в логах. Тестирование данного канала в таких условиях будет невозможно, если только вы не используете VPN, который обходит эти ограничения.
